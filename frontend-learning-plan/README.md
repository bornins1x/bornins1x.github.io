# Java 后端转全栈：React + TypeScript 30 天实战计划

适用对象：Java 后端开发，前端 0 基础。  
总目标：30 天后能独立完成一个 React + TypeScript 中后台小项目，并能和 Java 后端接口联调。  
建议投入：每天 2-3 小时。

## 如何使用这套计划

每周只打开当前周文档执行。每天按这个节奏推进：

```txt
20-30 分钟：看当天指定教程
90-120 分钟：完成当天实战
10-20 分钟：写当天学习日志
```

学习时不要做长笔记，只记录：

- 今天学了什么
- 今天写了什么
- 今天卡住的问题
- 明天要补什么

每日笔记模板见：[daily-note-template.md](./daily-note-template.md)

## 30 天最终产出

完成后你应该有 3 个项目：

1. 商品管理小工具
2. 用户管理后台
3. 任务管理系统雏形

你应该掌握：

- HTML 页面结构
- CSS 基础样式和 Flex 布局
- JavaScript 数组、对象、函数、异步请求
- TypeScript 业务类型建模
- React 组件、状态、事件、表单、Effect
- React Router 路由
- Axios 请求封装
- TanStack Query 请求状态管理
- Ant Design 中后台组件
- 登录鉴权和路由守卫
- CRUD 页面开发
- Java 后端接口联调

## 学习原则

- 每天都要写代码，不要连续看教程超过 2 天。
- 先会用，再理解原理。
- 优先学习中后台高频能力：布局、组件、状态、表单、路由、接口请求、登录鉴权、CRUD。
- 前 30 天不要深挖 Webpack、复杂动画、浏览器内核、组件库源码。
- 遇到报错先看浏览器控制台和终端日志，再查文档。
